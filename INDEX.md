## üìã QUICK DEPLOYMENT STEPS

### 5-Step Express Deploy

```bash
# Step 1: Prepare
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/USERNAME/UberApplication.git
git push -u origin main

# Step 2: Access
# Visit: https://labs.play-with-docker.com/
# Click: "+ ADD NEW INSTANCE"

# Step 3: Clone
git clone https://github.com/USERNAME/UberApplication.git
cd UberApplication

# Step 4: Deploy
docker-compose up -d

# Step 5: Access
# Click port 8080 link
# Visit: /swagger-ui.html
```

## ‚úÖ VERIFICATION QUICK CHECKS

After deployment, verify:

```bash
# Check 1: Services running
docker-compose ps
# Expected: Both containers "healthy"

# Check 2: Application ready
curl http://localhost:8080/actuator/health
# Expected: {"status":"UP"}

# Check 3: Swagger UI
# Visit: http://localhost:8080/swagger-ui.html
# Expected: API documentation visible

# Check 4: Database
docker exec -it uber-postgres psql -U uber -d uberdb -c "\dt"
# Expected: Table list (may be empty)
```

## üö® EMERGENCY QUICK FIXES

| Problem | Quick Fix |
|---------|-----------|
| Build fails | `docker-compose build --no-cache` |
| Won't start | `docker-compose logs -f springboot-app` |
| Can't access | Wait 60s, then `docker-compose restart` |
| Port conflict | Edit docker-compose.yml, change port |
| Database error | `docker-compose restart postgres` |
